介绍一下我的新宠：yarn

发表于 2017-12-15 | 分类于 前端

枯燥的代码之外总能遇到一些让人兴奋的小东西。
前端的道路虽然漫长，但是走着走着总能遇到惊喜。
从将就着用一用的DW，到想要什么就有什么的Visual Studio Code。
从直接点开html到browser-sync。
从第一次知道npm(那匹马)还有npm i -S这种神奇的操作到yarn的普及。
在这个寒冷的冬天，还是能够感受到一丝丝的温暖的关怀

yarn 看喵星人如何统治地球

  之前总是或多或少听说过yarn，但是第一次用还是Angular5发布后好像只能用yarn安装，所以抱着试一试的态度（经典广告桥段）安装了。

  通常用的最多的npm i成功迁移到yarn,没错，这样就行了。

  npm i packge-name -S写成yarn add packge-name -S没毛病。

  npm start to yarn start

  npm run dev to yarn run dev

  npm run build to yarn build

  知道这些就可以愉快的开始啦，还有不知道的官网看看文档吧。cli并没有方便多少，但是不被发觉的贴心才能体现一个暖男的气质。

并行的安装

  这恐怕也是最直观的体验了吧。通常npm安装依赖是一个个来，这时你可以安心的倒杯水，看看别的或者歇一歇，水喝完了，你还可以赶上安装的最后冲刺。
yarn感觉就解决了一个长久以来的痛点，即便是我的渣渣电脑，yarn的安装比改过淘宝的npm还快上不少，你可以安心的看着进度条跑完。

很少被发现的好处

  学一些东西的时候可能会启动多个相同的项目，两个vue或者react。当你不知道怎么做时可能莫名奇妙就打开了另一个项目然后端口占用报错，后来就学乖了先关掉一个再打开另一个。但是yarn可以解决这个痛点，比如两个localhost:3000，另一个会以localhost:3001运行。项目有多少就能跑多少。

清晰的安装提示与报错

  再也不用大段大段的滚鼠标啦，本来不多的头发也能晚点掉光啦

吹完了

  从第一次跑起vue-cli的兴奋开始,npm就一直是一个让我困惑的存在。渐渐理解前端以后才发现在这支离破碎的生态中唯一要坚持的始终是你的代码。或许以后的项目没法完全抛弃npm，但至少我能够淡然面对一切抛出的错误并且偷偷把yarn开开，也可能压根不需要依赖这玩意儿，毕竟前端的事谁说的定呢？
